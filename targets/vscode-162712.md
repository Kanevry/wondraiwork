## Problem

VS Code does not respect the XDG Base Directory Specification on Linux. Configuration, data, and
cache files are stored in `~/.vscode/` and `~/.config/Code/` instead of using `$XDG_CONFIG_HOME`,
`$XDG_DATA_HOME`, and `$XDG_CACHE_HOME` when set. This breaks the expectations of Linux users who
organize their dotfiles according to XDG conventions and clutters the home directory.

## Impact

Affects all VS Code users on Linux who follow XDG conventions. With 183K stars, VS Code is the most
popular code editor. The "help wanted" label indicates the team is open to external contributions.

## Approach

1. Map all locations where VS Code writes to `~/.vscode/` and `~/.config/Code/`
2. Identify the path resolution utilities (likely in `src/vs/base/common/`)
3. Add XDG environment variable support with fallback to current defaults
4. Start with a single subsystem (e.g., extensions directory) as a proof-of-concept
5. Ensure backward compatibility: existing installations should not break

## Notes

This is a large effort if done completely but could be broken into incremental PRs. The "help
wanted" label is encouraging. The main risk is scope: VS Code stores data in many locations, and a
complete XDG migration requires touching multiple subsystems. Consider starting with a focused PR
that adds XDG support for one specific data category and getting maintainer feedback on the
approach.
