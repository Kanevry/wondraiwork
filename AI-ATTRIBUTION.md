# AI Attribution Policy

> Transparent Disclosure of AI Tool Usage in WondrAIWork Contributions

## Purpose

WondrAIWork is a framework for systematic open-source contribution. Because AI tools play a role in
the workflow, transparency about their use is not optional — it is a core requirement.

Open-source communities are built on trust. Contributors trust each other to submit work they
understand and stand behind. Maintainers trust that contributions are genuine and well-considered.
Disclosing AI assistance preserves that trust and sets an honest foundation for collaboration.

This policy exists to ensure that every contribution made through WondrAIWork is transparent about
how it was produced.

## Policy

### In This Repository

All commits in this repository must disclose AI assistance using standardized Git trailers:

- Use `Co-Authored-By` when work was done collaboratively with an AI tool.
- Use `Assisted-By` or `Generated-By` trailers as described in the Attribution Format section below.

No commit that involved AI assistance should omit attribution.

### In Target Repositories

Before submitting a contribution to any target repository:

1. Check whether the target repo has an explicit AI contribution policy.
2. If the target repo has a policy, follow it exactly.
3. If no policy exists, disclose AI tool usage in the pull request description. A brief, factual
   statement is sufficient — for example: "AI tools (Claude Code) were used to assist with
   implementation and testing."

Never misrepresent AI-generated or AI-assisted work as purely human-authored.

## Attribution Format

Use the following trailers in commit messages, choosing the level that accurately reflects the
degree of AI involvement:

### Minor AI Assistance

For code suggestions, refactoring help, boilerplate generation, or similar incremental assistance:

```
Assisted-By: Claude Code
```

### Substantial AI-Generated Content

For code, documentation, or other content where the AI tool produced the majority of the output:

```
Generated-By: Claude Code
```

### Model and Version

When the specific model or version is known, include it for precision:

```
Assisted-By: Claude Code (Claude Opus 4, 1M context)
```

```
Generated-By: Claude Code (Claude Opus 4, 1M context)
```

Use the most specific identifier available at the time of the contribution.

## Human Responsibility

AI tools do not bear responsibility for contributions. The human contributor does.

- **Understand**: The contributor must fully understand every line of code before submission. If you
  cannot explain why a change works, do not submit it.
- **Review**: All AI output must be critically reviewed. AI tools produce plausible but sometimes
  incorrect code. Blind acceptance is not permitted.
- **Test**: Run the target repository's full test suite. Verify that the contribution works as
  intended. Do not rely on AI assertions that the code is correct.
- **Own**: The human contributor takes full responsibility for the contribution, including any bugs,
  regressions, or issues that arise from it.

AI output is a starting point, not a finished product.

## What Must Be Human-Written

Certain parts of the contribution process require authentic human engagement and cannot be delegated
to AI:

### Pull Request Descriptions

PR descriptions may be drafted with AI assistance, but the final submitted text must be
human-reviewed and edited. The contributor must ensure the description accurately represents their
understanding of the change, not a summary generated by a tool.

### Review Responses

All responses to maintainer feedback and code review comments must be authentic human engagement.
Maintainers are investing their time in reviewing the contribution; they deserve genuine
interaction, not generated replies.

### Issue Analysis and Evaluation

The discovery, evaluation, and scoring of target issues (as defined in the WondrAIWork methodology)
must reflect human judgment. AI tools may assist with research, but the strategic decisions — which
issues to pursue, how to score them, whether to proceed — are human decisions.

## Target Repo Policies

The open-source ecosystem is actively developing norms around AI-assisted contributions. Policies
vary widely between projects.

### Before Every Contribution

1. Check the target repository for an AI contribution policy. Look in `CONTRIBUTING.md`,
   `CODE_OF_CONDUCT.md`, `.github/`, or repository documentation.
2. Search for prior discussion in the repository's issues or discussions about AI-generated
   contributions.
3. Consult the community-maintained list of open-source AI contribution policies:
   [github.com/melissawm/open-source-ai-contribution-policies](https://github.com/melissawm/open-source-ai-contribution-policies)

### Respect Restrictions

If a target repository explicitly bans AI-assisted or AI-generated contributions, respect that
policy completely. Do not submit AI-assisted work to repositories that prohibit it, regardless of
how minor the assistance was. There are no exceptions.

### When Policies Are Ambiguous

If a repository's policy is unclear, err on the side of disclosure. Mention AI tool usage in the PR
description and let the maintainers decide whether to proceed.

## Philosophy

Transparency builds trust. Concealing AI usage — even when no explicit policy forbids it —
undermines the collaborative foundation of open source.

AI is a tool, not a replacement for human judgment, expertise, or accountability. It accelerates
certain tasks, but it does not substitute for understanding the problem, the codebase, or the
community.

Quality matters more than speed. A well-understood, thoroughly tested contribution submitted a day
later is worth more than a hastily generated one submitted within the hour. The goal is to make
meaningful contributions to open-source projects, not to maximize output volume.

Every contribution bearing the WondrAIWork name should reflect this standard.
