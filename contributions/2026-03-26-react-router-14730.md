### Codebase Analysis

Root cause identified in `packages/react-router-dev/vite/plugin.ts`:

- Line 4063: `let defaultExternalConditions = ["node"]` hardcoded for ALL SSR environments
- Introduced in PR #13871 as conscious fix (Vite does not export `defaultExternalConditions`)
- Correct for Node.js, but breaks Cloudflare Workers and other edge runtimes

### Implementation

Two-part fix in `plugin.ts` (19 insertions, 2 deletions):

1. **`getBaseServerOptions()`**: When `v8_viteEnvironmentApi` is true, `externalConditions` no
   longer includes `"node"`. This is set per-environment instead.

2. **`configEnvironment` hook**: `"node"` is only added to `externalConditions` when
   `options.resolve?.noExternal !== true`. The `noExternal: true` signal is set by
   `@cloudflare/vite-plugin` for non-Node runtimes.

## Key Learnings

### Technical

1. Vite 5->6 changed condition handling: custom conditions now REPLACE defaults instead of merging
2. `noExternal: true` is the reliable signal for non-Node runtimes in Vite's architecture
3. `configEnvironment` hook runs per-environment AFTER `config` hook -- enables runtime-specific
   logic
4. Cloudflare's `@cloudflare/vite-plugin` sets `noExternal: true` in its `config` hook
